# metrics end 与异常处理

本文件仅在 `end` 失败、子任务异常退出或需要判断是否补报时读取。

## end 失败处理

1. **schema 校验失败**：只修复 `metrics.json` 中字段类型 / 缺失业务字段，然后重新执行 `end`；严禁重新执行 `start`。
2. **网络 / JWT 问题**：重新获取 JWT 后重试 `end`，最多 3 次；严禁重新执行 `start`。
3. **仍失败**：保留 `metrics.json` 与错误信息，在任务状态中记录失败原因。

## 异常退出

子任务崩溃 / 用户中断 / 未走到 `end` 时，不要补调 `end`；丢弃本次 `metrics.json`，下次子任务正常进入新一轮 `start → end`。
